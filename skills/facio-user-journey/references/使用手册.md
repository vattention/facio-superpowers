# 用户行为动线分析 · 使用手册

给市场 / 产品同学。**不需要写代码，不需要生产环境权限，不需要懂数据库。**

你要做的只有两件事：装一次，然后用大白话提问。

---

## 一、一次性设置（约 5 分钟，只做一次）

**Claude Code 和 Codex 都能用。** 第 1 步一样，第 2 步按你用哪个选。

### 1. 装 skill（两边通用）

在终端里粘这一行：

```bash
npx @vattention/facio-superpowers init
```

它会把团队的 skills 同时装给 Claude Code 和 Codex，两边都认。装完就一直在。

### 2. 连 Mixpanel（按你用的工具二选一）

**如果你用 Claude Code：**

```bash
claude mcp add --transport http mixpanel https://mcp.mixpanel.com/mcp
```

然后输入 `/mcp` → 选 `mixpanel` → 点 **Authenticate** → 浏览器里登录授权。

**如果你用 Codex：**

```bash
codex mcp add mixpanel --url https://mcp.mixpanel.com/mcp
```

这一条就够了 —— 它会自己发现需要授权、直接弹出登录链接，浏览器里点一下就好。

> 万一后来掉线了要重新授权：`codex mcp login mixpanel`

### 3. 重启一下

退出 Claude Code / Codex，重新打开。**这一步不能省** —— 新装的东西要重启才认。

> **如果授权失败**：多半是你们 Mixpanel 组织管理员还没开启 MCP 功能。找管理员开一下，不是你的问题。

## 一点五、以后怎么升级（重要）

`init` 只装**你还没有的东西**，已经装过的它不会覆盖。所以**升级不能再跑 `init`，要跑 `sync`**：

```bash
npx @vattention/facio-superpowers@latest sync
```

Claude Code 和 Codex 两边都会一起更新。

> **什么时候该跑？** 听说 skill 更新了、或者发现行为跟这份手册对不上时。跑一下没有副作用。

## 二、怎么用

**就用大白话问。** 不用记命令，不用背参数。

> 「分析一下 xxx@gmail.com 这个邀测用户」
>
> 「查一下这个邮箱的行为动线」
>
> 「这个用户到底有没有做出东西？」
>
> 「他是不是流失了？」

Claude 会自己认出该用这个 skill，然后去查、去分析、最后给你一个 HTML 报告，双击就能看。

**想要能发给别人的版本**，加一句：

> 「出一份脱敏版」

---

## 三、它能回答什么

| 你想知道 | 能答吗 |
|---|---|
| 他做出成品了吗？导出成功过几次？ | ✅ **这是最硬的指标** |
| 他现在还在用吗？还是真的走了？ | ✅ |
| 他哪天来的、来了几次、每次干了多久 | ✅ |
| 每天投入多少 vs 产出多少（趋势） | ✅ |
| 用了多少次 AI、AI 有没有失败 | ✅ |
| 手动改了多少次、导入了多少素材 | ✅ |
| 邀请码有没有激活、卡在哪一步 | ✅ |
| **他为什么不满意 / 具体卡在哪句话** | ❌ 需要研发（见第五节） |
| 花了多少积分 | ❌ 需要研发 |

---

## 四、报告怎么读

### 先看这三个数

**导出成功 / 尝试** —— 最重要的一个。有导出＝这人真的把东西做出来并发出去了，激活成功。`0` 就是还没产出过成品。

**AI Agent 任务完成** —— 比如 `30 / 30` 表示发起 30 次、完成 30 次。分母分子不相等＝AI 中途卡住了。

**最后活跃** —— 就是字面意思。如果是今天或这两天，**这人没走**，别写成流失。

### 再看「每日投入 vs 产出」那张表

两条线一起看，能一眼分出三种完全不同的情况：

| 形态 | 含义 | 该做什么 |
|---|---|---|
| 投入涨，导出也涨 | 健康 | 不用管 |
| **投入涨，导出一直是 0** | **在拼命干但发不出去** | 最值得追，问题通常在体验里 |
| 两条都归零，且早于最后活跃 | 真流失了 | 复盘获客/预期 |
| 导出过一次，之后再没有 | 首次成了，复用断了 | 查第二次为什么没成 |

### 几个容易误读的地方

- **「AI 任务 30/30 全完成」不等于用户满意。** 工具全部返回成功，用户照样可能觉得成片很烂。绿灯只说明程序没报错，不代表东西做得好。
- **报告里的时间统一是 UTC**，已经换算过了，别再自己加减时差。
- **`—` 表示这项要研发权限才能查**，不是数据缺失，也不是出错了。

## 五、碰到「为什么」问不下去时

Skill 会明确告诉你哪些是它查不到的。这不是它坏了，是**设计如此**：

Mixpanel 记录的是**发生了什么**（点了什么、成没成功），不记录**用户说了什么**。所以「他为什么不满意」「具体哪一步惹毛他了」这类问题，答案在他跟 AI 的对话原文里，那需要生产数据库权限。

**这时候找研发**，把邮箱给他们，说：「帮我跑一下 facio-user-journey 的 Tier 2，我想看他自己说了什么」。他们跑出来的报告会多两块内容：AI 对话细节和积分明细。

> ⚠️ **不要去申请生产数据库权限。** 这个 skill 特意设计成不需要你有那个权限，你也不应该有。

## 六、隐私（这条必须遵守）

报告里有**真实用户的邮箱、IP、以及他跟 AI 说过的原话**。

- ✅ 留在你自己电脑上看
- ✅ 要发给别人，先要一份**脱敏版**（邮箱会变成 `j***@gmail.com`）
- ❌ **不要**上传到任何在线文档、网盘、外部工具
- ❌ **不要**贴进公开群

**脱敏版有个边界你要知道**：它会遮掉邮箱、账号 ID、IP，但**不会**遮掉报告里引用的用户原话 —— 因为那些话往往正是报告的价值所在。所以**发之前自己扫一眼**：那些引用里有没有带出人名、频道名、文件路径、链接。**你引用了什么，你负责。**

## 七、常见问题

**Q：输入 `/mcp` 看不到 mixpanel？（Claude Code）**
没重启。退出 Claude Code 重开。

**Q：Codex 里怎么确认连上了？**
跑 `codex mcp list`，看 `mixpanel` 那行的状态。

**Q：授权点了没反应 / 报错？**
找 Mixpanel 组织管理员确认 MCP 功能已开启。

**Q：跑出来的结果跟这份手册对不上？**
你的版本旧了。跑 `npx @vattention/facio-superpowers@latest sync`（注意是 `sync` 不是 `init` —— `init` 不会覆盖已装的东西）。

**Q：查出来说「这个用户没有任何事件」？**
两种可能：邮箱不对（问问他实际注册用的哪个），或者这人被邀请了但从没真正注册进来过。后者的名单在运营后台，不在这里。

**Q：报告能改吗？我想加点东西？**
直接跟 Claude 说要加什么就行。

**Q：能一次分析一批用户吗？**
现在是单个用户。有需求跟研发提。

**Q：数据是实时的吗？**
Mixpanel 基本实时（分钟级）。

## 八、一句话总结

装一次 → 大白话问 → 拿 HTML 报告 → **先看「导出成功」和「最后活跃」** → 想问「为什么」就找研发要 Tier 2 → 发出去之前先脱敏。
