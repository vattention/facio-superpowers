# 练习项目

通过完成这个练习项目，你将亲身体验 Facio Superpowers 的完整工作流程。

## 练习目标

完成这个练习后，你将能够：
- [x] 使用 `/prepare-context` 准备上下文
- [x] 通过自然对话创建和讨论 Context
- [x] 参与需求澄清和验收标准制定
- [x] 体验任务拆分和 TDD 开发流程
- [x] 使用 `/verification-before-completion` 完成验证

---

## 练习场景

### 背景

假设你加入了一个"待办事项"应用的开发团队。产品经理提出了一个新需求：

> "用户希望能给待办事项设置优先级，这样可以更好地管理任务。"

你的任务是完整走一遍从需求讨论到开发完成的流程。

---

## 练习步骤

### 第一部分：需求讨论（约 15 分钟）

#### 步骤 1：启动 Claude Code

```bash
cd 你的练习项目目录
claude
```

#### 步骤 2：提出需求

输入：
```
我想给待办事项加一个优先级功能
```

**观察**：
- AI 会询问是否创建 Context
- 回答"是"

#### 步骤 3：参与需求讨论

AI 会问你一系列问题，比如：
- "优先级分几级？高中低三级还是数字？"
- "默认优先级是什么？"
- "优先级可以修改吗？"

**练习要求**：
- 认真回答每个问题
- 如果 AI 提供选项，选择其中一个
- 如果有自己的想法，可以补充

#### 步骤 4：确认验收标准

当 AI 给出验收标准时，仔细阅读：

```
验收标准：
- [ ] 创建待办时可以选择优先级（高/中/低）
- [ ] 默认优先级为"中"
- [ ] 待办列表按优先级排序显示
- [ ] 可以修改已有待办的优先级
- [ ] 优先级用颜色区分（红/黄/绿）
```

确认后说：
```
验收标准确认
```

#### 步骤 5：观察测试用例生成

AI 会生成测试用例：
```
测试用例：
- TC1: 创建高优先级待办，显示红色标记
- TC2: 创建待办不选优先级，默认为中
- ...
```

确认后说：
```
测试用例确认，可以开始开发了
```

---

### 第二部分：技术讨论（约 10 分钟）

#### 步骤 6：进入 Task 阶段

说：
```
我来开发这个功能
```

AI 会进入技术讨论模式。

#### 步骤 7：准备上下文

输入：
```
/prepare-context
```

观察 AI 搜索相关文档和决策。

#### 步骤 8：技术方案讨论

AI 会询问技术细节：
- "数据库如何存储优先级？枚举还是数字？"
- "前端用什么组件展示优先级？"

跟随 AI 的引导，完成技术方案讨论。

#### 步骤 9：确认任务拆分

AI 会将功能拆分成小任务：
```
任务拆分：
1. 数据库添加 priority 字段
2. API 支持优先级参数
3. 待办列表按优先级排序
4. 前端添加优先级选择器
5. 编写测试用例
```

确认后，AI 开始实现。

---

### 第三部分：观察开发过程（约 10 分钟）

#### 步骤 10：观察 TDD 流程

AI 会按照 TDD 流程：
1. 先写测试（会显示测试代码）
2. 运行测试（显示失败）
3. 写实现代码
4. 运行测试（显示通过）

**注意观察**：
- AI 是如何组织代码的
- 测试用例覆盖了哪些场景

#### 步骤 11：代码审查

每个任务完成后，检查代码：
- 如果有问题：`这里不太对，xxx`
- 如果满意：`继续`

---

### 第四部分：完成验证（约 5 分钟）

#### 步骤 12：验证完成

所有任务完成后，输入：
```
/verification-before-completion
```

观察 AI：
- 运行测试套件
- 检查代码覆盖率
- 确认文档更新

#### 步骤 13：查看结果

```
总结一下这次开发完成了什么
```

## 常见问题

### Q: 练习用什么项目？

可以使用任何前端/后端项目，或者创建一个简单的练习项目：
```bash
mkdir todo-practice
cd todo-practice
npm init -y
```

### Q: 不熟悉某个技术栈怎么办？

告诉 AI：
```
我不熟悉 React，请详细解释一下
```

AI 会给出更详细的说明。

### Q: 练习中途出错了怎么办？

直接告诉 AI：
```
重新开始练习
```
