# 测试人员指南

本指南帮助测试人员了解如何使用 Facio Superpowers 参与测试用例设计、功能测试和验收。

## 测试人员的核心工作

在 Facio Superpowers 中，测试人员主要负责：

1. **参与测试用例设计**（Context 阶段）
2. **执行功能测试**（Task 阶段）
3. **Bug 反馈与跟踪**
4. **协助验收**

---

## 一、测试用例设计

### 何时参与？

当产品和 AI 完成需求讨论、确认验收标准后，会进入"测试用例互搏"阶段。这时候你可以参与。

### 查看待测试的需求

```
有哪些需求在等待测试用例设计？
```

### 查看已有的测试用例

```
看看收藏夹功能的测试用例
```

AI 会展示：
```markdown
# 测试用例

## 正向场景
- [ ] TC1: 用户点击收藏按钮，按钮变为"已收藏"状态
- [ ] TC2: 用户查看收藏列表，显示所有已收藏内容
- [ ] TC3: 用户取消收藏，内容从列表移除

## 边界场景
- [ ] TC4: 收藏 100 个内容后继续收藏
- [ ] TC5: 收藏内容被删除后，列表显示处理

## 异常场景
- [ ] TC6: 网络断开时点击收藏
- [ ] TC7: 未登录时点击收藏
```

### 补充测试用例

发现遗漏的场景：
```
还需要测试：连续快速点击收藏按钮的情况
```

AI 会将你的补充加入测试用例文档。

### 测试用例的重要性

在 Facio Superpowers 中，测试用例有特殊地位：

1. **需求决策的前置条件**：没有测试用例，需求不能进入"已决策"状态
2. **开发的指导文档**：开发按照测试用例实现
3. **验收的检查清单**：验收时逐项核对

## 二、执行功能测试

### 查看待测试的任务

```
有哪些任务是待测试状态？
```

### 开始测试

```
我来测试"收藏夹功能"
```

AI 会：
1. 展示测试用例清单
2. 引导你逐项执行测试
3. 记录测试结果

### 记录测试结果

**测试通过**：
```
TC1 通过
TC2 通过
```

**测试不通过**：
```
TC3 不通过，问题是：取消收藏后列表没有刷新
```

AI 会创建 Bug 记录，关联到相关任务。

### 完成测试

```
收藏夹功能测试完成，有 2 个问题需要修复
```

任务状态会保持"待测试"直到问题修复。

## 三、Bug 反馈与跟踪

### 如何报告 Bug

直接描述问题：

```
发现一个 bug：在收藏列表页面，下拉刷新后收藏数量显示错误
```

AI 会引导你补充：
- 复现步骤
- 预期结果
- 实际结果
- 测试环境

### 完整的 Bug 报告示例

```
bug 报告：

问题：下拉刷新后收藏数量显示错误

复现步骤：
1. 进入收藏列表页面
2. 当前显示"共 10 个收藏"
3. 下拉刷新
4. 显示变成"共 0 个收藏"，但列表内容还在

预期结果：刷新后数量正确显示
实际结果：数量显示为 0
测试环境：iOS 17，iPhone 15
```

### 跟踪 Bug 状态

```
收藏功能相关的 bug 修复了吗？
```

### 验证 Bug 修复

```
帮我验证一下"收藏数量显示错误"这个 bug
```

AI 会提示你按照原来的复现步骤测试。


## 六、测试用例编写规范

### 好的测试用例特征

| 要素 | 说明 | 示例 |
|-----|------|------|
| 前置条件 | 测试开始前的状态 | "用户已登录" |
| 操作步骤 | 具体的操作动作 | "点击收藏按钮" |
| 预期结果 | 期望发生什么 | "按钮变为已收藏状态" |
| 可验证 | 结果可以明确判断 | 是/否，而非"感觉好" |

### 测试用例分类

**正向场景**：正常使用路径
```
- [ ] TC1: 已登录用户收藏一篇文章，收藏成功
```

**边界场景**：极限情况
```
- [ ] TC2: 收藏第 1000 个内容（假设上限 1000）
- [ ] TC3: 收藏标题超长（200 字符）的内容
```

**异常场景**：错误处理
```
- [ ] TC4: 网络断开时收藏，显示友好提示
- [ ] TC5: 收藏已被删除的内容，显示"内容不存在"
```

**兼容性场景**：不同环境
```
- [ ] TC6: iOS Safari 浏览器正常使用
- [ ] TC7: Android Chrome 浏览器正常使用
```

## 七、与开发协作

### 测试发现的问题分级

| 级别 | 定义 | 处理 |
|-----|------|------|
| P0 | 阻塞性问题，无法继续 | 立即修复 |
| P1 | 严重问题，核心功能受影响 | 上线前必须修复 |
| P2 | 一般问题，有规避方案 | 计划修复 |
| P3 | 轻微问题，体验优化 | 有空修复 |

### 问题描述要清晰

**不好的描述**：
```
收藏功能有问题
```

**好的描述**：
```
问题：收藏按钮点击后无反应
复现：登录后进入文章详情页，点击收藏按钮
频率：必现
环境：Chrome 120，Windows 11
```

## 八、常见问题

### Q: 测试用例应该多细？

原则：能让任何人按照用例操作并判断结果。

如果操作需要专业知识才能理解，就需要写得更细。

### Q: 开发说"这不是 bug，是特性"

```
需要确认一下：xxx 这个行为是设计如此还是 bug？
```

AI 会帮你查找相关的需求讨论记录。

### Q: 时间紧，来不及测所有用例

```
收藏功能的核心用例有哪些？
```

AI 会帮你识别优先级高的测试用例。

### Q: 测试环境有问题

```
测试环境的 xxx 服务挂了
```

AI 会记录并通知相关人员。

## 九、测试报告模板

完成测试后，可以生成测试报告：

```
生成收藏功能的测试报告
```

AI 会生成：

```markdown
# 测试报告：收藏功能

## 测试概要
- 测试时间：2026-03-10
- 测试人：[你的名字]
- 测试版本：v0.5.0

## 测试结果
- 总用例数：15
- 通过：13
- 失败：2
- 跳过：0

## 失败用例
| 用例 | 问题描述 | 级别 |
|-----|---------|------|
| TC5 | 取消收藏后列表未刷新 | P1 |
| TC8 | 网络断开提示不友好 | P2 |

## 测试结论
收藏功能基本可用，有 2 个问题需要修复后才能上线。

## 遗留风险
- 未测试：超过 1000 个收藏的性能表现
```
