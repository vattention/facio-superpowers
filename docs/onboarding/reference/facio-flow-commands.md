# Facio Flow 命令速查

Facio Flow 是团队协作工具，用于管理需求讨论、任务跟踪和版本规划。

## 概念速览

| 概念 | 说明 | 类比 |
|-----|------|------|
| Context | 需求讨论的容器 | 会议纪要 |
| Task | 具体的开发任务 | 工单 |
| Version | 版本规划 | 里程碑 |
| Artifact | 附件（如 spec、设计稿） | 附件 |

---

## Context（上下文）操作

### 创建 Context

**场景**：有新的需求想法时

**说法**：
```
我想做一个收藏夹功能
```
AI 会询问是否创建 Context。

**或直接创建**：
```
创建一个讨论：收藏夹功能
```

---

### 查看 Context 列表

**场景**：查看当前有哪些讨论

**说法**：
```
看看有哪些正在讨论的需求
列出所有 open 状态的 Context
```

**筛选**：
```
看看 video-editor 产品的讨论
看看已决策待认领的需求
```

---

### 读取 Context 内容

**场景**：了解某个讨论的详情

**说法**：
```
看看收藏夹功能的讨论
读取 CTX-012 的内容
```

---

### 追加讨论内容

**场景**：在讨论中补充信息

**说法**：
```
补充一下：收藏需要支持分组功能
```

AI 会自动判断是否需要追加到 Context。

---

### 记录决策

**场景**：讨论结束，确定方案

**说法**：
```
就按方案 A 做
验收标准确认了
```

AI 会记录决策，包括：
- 决策内容
- 决策原因
- 被否决的方案

---

### 认领 Context

**场景**：开始开发已决策的需求

**说法**：
```
我来做收藏夹功能
认领 CTX-012
```

---

### 创建 Followup

**场景**：已完成的需求需要修改

**说法**：
```
收藏夹功能需要改一下，要加分组
```

AI 会创建新的 Context 并关联原讨论。

---

### 关闭 Context

**场景**：需求已完成或不再需要

**说法**：
```
收藏夹功能做完了，关闭讨论
```

---

### 取消 Context

**场景**：需求取消不做了

**说法**：
```
收藏夹功能不做了
取消 CTX-012
```

---

### 拆分 Context

**场景**：需求太大，要拆成多个小需求

**说法**：
```
这个需求太大了，帮我拆分一下
```

---

## Task（任务）操作

### 查看任务列表

**说法**：
```
看看现在有哪些任务
我有哪些任务
```

**筛选**：
```
看看待开发的任务
看看 v0.5 版本的任务
看看小王的任务
```

---

### 创建任务

**说法**：
```
创建一个任务：修复登录闪退问题
```

**带关联版本**：
```
创建任务"优化首页性能"，关联到 video-editor v0.5
```

---

### 认领任务

**说法**：
```
我来做"修复登录闪退"这个任务
认领 TASK-023
```

---

### 更新任务状态

**说法**：
```
登录闪退修好了
TASK-023 开发完成
```

**状态流转**：
- 待开发 → 开发中 → 待测试 → 待验收 → 已完成

---

### 更新任务进度

**说法**：
```
登录优化做了一半了
TASK-023 进度 60%
```

---

### 查看待认领任务

**说法**：
```
有哪些可以认领的任务？
```

---

**说法**：
```

```

## Artifact（附件）操作

### 保存附件

**场景**：AI 输出了完整的 spec、设计等

**说法**：
```
保存这份 spec
```

**附件类型**：
- `spec`：需求规格
- `design`：技术设计
- `test-cases`：测试用例
- `interaction`：交互设计
- `proposal`：提案

---

### 查看附件列表

**说法**：
```
看看收藏夹功能有哪些附件
```

---

### 读取附件内容

**说法**：
```
看看收藏夹功能的 spec
```

---

### 同步 Spec 到全局目录

**说法**：
```
把这份 spec 同步到 video-editor/specs/ 目录
```

## Context 状态流转

```
open（讨论中）
    ↓ 记录决策
decided（已决策）
    ↓ 认领开发
claimed（已认领）
    ↓ 开发完成
closed（已完成）

特殊状态：
- cancelled：已取消
- split：已拆分为子需求
```

---

## Task 状态流转

```
待开发
    ↓ 开始开发
开发中
    ↓ 开发完成
待测试
    ↓ 测试通过
待验收
    ↓ 验收通过
已完成
```

---

## 注意事项

1. **自动记录**：大部分操作 AI 会自动判断并记录，不需要显式命令
2. **自然语言**：用自然语言描述即可，不需要记忆精确命令
3. **上下文感知**：AI 会根据当前对话判断操作哪个 Context/Task
4. **团队可见**：所有记录团队成员都可以看到
