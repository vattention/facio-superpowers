# 常见问题

本文档收集使用 Facio Superpowers 时的常见问题和解决方案。

---

## 安装和环境问题

### Q: `claude` 命令找不到

**症状**：
```
zsh: command not found: claude
```

**解决方案**：
1. 确认已安装 Claude Code：
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```
2. 检查 npm 全局路径是否在 PATH 中：
   ```bash
   npm bin -g
   # 确认这个路径在你的 PATH 中
   ```
3. 重新打开终端窗口

---

### Q: Superpowers 没有自动加载

**症状**：AI 没有使用 Skill，像普通聊天一样

**解决方案**：
1. 确认项目已初始化 Superpowers：
   ```bash
   ls -la ~/.claude/skills/
   # 应该能看到 superpowers 相关文件
   ```
2. 重新初始化：
   ```bash
   npx @vattention/facio-superpowers init
   ```
3. 重启 Claude Code 会话

---

### Q: Facio Flow 连接失败

**症状**：操作 Context/Task 时报错

**解决方案**：
1. 检查 MCP 配置是否正确
2. 确认 facio-flow 服务正在运行
3. 检查网络连接

---

## 使用问题

### Q: AI 问太多问题，我想直接开始写代码

**解决方案**：

告诉 AI：
```
这是个简单改动，跳过设计讨论，直接实现
```

或者：
```
先按默认方案做，有问题再调整
```

**提醒**：对于复杂功能，设计讨论能避免后期返工。

---

### Q: AI 使用了错误的 Skill

**解决方案**：

直接指定：
```
用 systematic-debugging 来排查这个问题
```

或者取消当前 Skill：
```
停下来，不需要用这个流程
```

---

### Q: AI 生成的代码风格和项目不一致

**解决方案**：

1. 临时指定：
   ```
   请按照项目现有的代码风格重写
   ```

2. 永久配置（推荐）：
   在项目 `CLAUDE.md` 中添加代码规范说明：
   ```markdown
   ## 代码风格
   - 使用 2 空格缩进
   - 组件使用 PascalCase 命名
   - 函数使用 camelCase 命名
   ```

---

### Q: 不想每次都跑 `/prepare-context`

**解决方案**：

对于简单修改：
```
这是个小改动，不需要查上下文
```

但对于重要功能，建议保持这个习惯，避免踩坑。

---

### Q: Context/Task 太多，找不到我要的

**解决方案**：

使用搜索：
```
搜索关于"收藏"的讨论
```

按状态筛选：
```
看看 open 状态的 Context
看看我负责的 Task
```

---

### Q: 讨论到一半意外中断了

**解决方案**：

Context 会自动保存讨论内容。下次说：
```
继续讨论收藏夹功能
```

AI 会读取之前的讨论，从上次的地方继续。

---

### Q: 提交代码时 hook 报错

**症状**：
```
pre-commit hook failed
```

**解决方案**：
1. 查看具体错误信息
2. 常见原因：
   - ESLint 检查不通过 → 修复代码风格
   - 测试失败 → 修复测试
   - 类型检查失败 → 修复类型错误
3. 修复后重新提交

---

### Q: AI 没有更新 Task 状态

**解决方案**：

手动提醒：
```
这个任务做完了，帮我更新状态
```

或直接说：
```
把"实现收藏 API"这个任务标记为已完成
```

---

## 协作问题

### Q: 看不到同事创建的 Context

**可能原因**：
1. 同事使用的是不同的 product
2. facio-flow 数据未同步

**解决方案**：
```
列出所有 Context，包括其他产品的
```

---

### Q: 需求已经做完了，但 Context 还是 open 状态

**解决方案**：

提醒关闭：
```
收藏夹功能已经做完了，帮我关闭这个 Context
```

---

### Q: 想把讨论交给其他人继续

**解决方案**：
```
把收藏夹功能的讨论交接给小王
```

AI 会调用 handoff 功能，并生成交接摘要。

---

## 性能问题

### Q: AI 响应很慢

**可能原因**：
1. 网络延迟
2. 代码库太大，搜索耗时

**解决方案**：
1. 检查网络连接
2. 缩小搜索范围：
   ```
   只看 src/components 目录的代码
   ```

---

### Q: `/prepare-context` 返回太多无关内容

**解决方案**：

更精确地描述：
```
查找和用户认证相关的 ADR 和讨论
```

---

## Git 问题

### Q: Git 冲突不知道怎么解决

**解决方案**：
```
帮我解决这个 Git 冲突
```

AI 会分析冲突并给出建议。

---

### Q: 不小心提交了敏感信息

**解决方案**：

**立即**告诉 AI：
```
我不小心把密码提交了，帮我处理
```

AI 会指导你：
1. 从历史记录中删除
2. 更换密码
3. 检查是否已推送到远程

---

### Q: 想撤销最近的提交

**解决方案**：
```
撤销最近一次提交，保留代码改动
```

AI 会执行合适的 git 操作。
